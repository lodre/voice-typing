# VoiceTyping — локальный голосовой ввод для Windows 11 (русский + английский)

Диктовка в любое окно: зажали **Ctrl+Ё** — говорите — отпустили — текст напечатался там,
где стоит курсор. Работает полностью офлайн (OpenAI Whisper large-v3 через faster-whisper),
язык определяется автоматически, пунктуация расставляется сама.

## Как пользоваться

1. Запустите `VoiceTyping.exe` (или `python app.py` из исходников).
   В трее появится кружок с микрофоном: серый — грузится модель (~3–5 с),
   зелёный — готов, красный — идёт запись, синий — распознавание.
2. Поставьте курсор в любое текстовое поле.
3. **Зажмите Ctrl+Ё, говорите, отпустите.** Высокий сигнал — запись пошла,
   низкий — распознаётся. Через мгновение текст появится у курсора.
4. Правый клик по значку в трее:
   - **Язык**: авто (RU/EN) / только русский / только английский —
     ручная фиксация помогает для речи «вперемешку» с иностранными словами;
   - **Автозапуск при входе в Windows** — галочка создаёт/убирает ярлык в автозагрузке;
   - **Выход**.

## Перенос на другой компьютер

Скопируйте папку `VoiceTyping` целиком (вместе с `models/`) куда угодно — на диск, флешку.
На новом ПК: запустить `VoiceTyping.exe`, при желании включить автозапуск в меню трея.
Ничего устанавливать не нужно.

- **С видеокартой NVIDIA (GTX 10xx и новее)** — распознавание меньше секунды.
- **Без NVIDIA** — автоматически переключится на процессор. large-v3 на CPU медленная;
  замените модель на `small` или `large-v3-turbo` (см. ниже).

## Настройки — `config.json` (рядом с exe)

| Ключ | По умолчанию | Что значит |
|---|---|---|
| `ptt_key_vk` | `192` (Ё/`) | vk-код клавиши push-to-talk ([коды](https://learn.microsoft.com/windows/win32/inputdev/virtual-key-codes)) |
| `ptt_modifiers` | `["ctrl"]` | модификаторы: `ctrl`, `alt`, `shift`, `win`; `[]` — без них |
| `language` | `auto` | `auto` / `ru` / `en` (дублируется меню в трее) |
| `model_path` | `models/faster-whisper-large-v3` | папка с моделью CTranslate2 |
| `device` | `auto` | `auto` (cuda → cpu) / `cuda` / `cpu` |
| `paste_method` | `clipboard` | `clipboard` (буфер+Ctrl+V) или `type_unicode` (посимвольно) |
| `restore_clipboard` | `true` | возвращать прежнее содержимое буфера после вставки |
| `input_device` | `null` | микрофон: `null` = системный по умолчанию |
| `beep` | `true` | звуковые сигналы записи |

После правки config.json перезапустите программу.

### Сменить модель (для слабых ПК)

```
python download_model.py Systran/faster-whisper-small          # ~0.5 ГБ, быстро на CPU
python download_model.py Systran/faster-whisper-large-v3-turbo # ~1.6 ГБ, компромисс
```

и пропишите путь в `model_path`. В собранной версии можно просто скопировать папку модели
в `models/` с другого ПК.

## Ограничения

- **Окна, запущенные от имени администратора**: Windows (UIPI) запрещает обычной программе
  вводить в них текст. Нужно — запускайте VoiceTyping тоже от администратора.
- **История буфера обмена (Win+V)**: продиктованный текст в ней остаётся, даже с
  `restore_clipboard` — так устроена вставка через буфер. Не устраивает — режим `type_unicode`.
- Сочетание Ctrl+Ё перехватывается программой полностью и до приложений не доходит
  (например, в VS Code перестанет открывать терминал, пока VoiceTyping запущен).

## Запуск из исходников (для разработки)

```
py -3.12 -m venv .venv
.venv\Scripts\pip install -r requirements.txt
.venv\Scripts\python download_model.py
.venv\Scripts\python app.py
```

Сборка переносимой папки: `build.ps1` (PyInstaller onedir + копирование модели).

## Если что-то не работает

Лог пишется в `voice-typing.log` рядом с программой.

- «Модель не найдена» — нет папки из `model_path`; скачайте модель.
- Нет звука сигналов/микрофон не пишет — проверьте микрофон по умолчанию в настройках Windows
  и разрешение на доступ к микрофону (Параметры → Конфиденциальность → Микрофон).
- Текст не вставился, но остался в буфере — нажмите Ctrl+V вручную; для этого приложения
  попробуйте `paste_method: "type_unicode"`.
